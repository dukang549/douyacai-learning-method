# 豆芽菜学习法

一个帮助 AI 从“解释知识”走向“帮助用户真正学会”的学习工作流。

它不是普通 prompt，也不只是一个回答模板。  
它的核心目标是：让 AI 在合适的时候不只负责讲解，还负责检验用户是否真的理解。

---

## 这是什么

豆芽菜学习法是一套面向 AI 学习助手的通用工作流，适用于：

- ChatGPT
- Claude
- Codex
- Grok
- 其他支持长提示词或工作流提示的 Agent 系统

它试图解决一个常见问题：

> 大多数 AI 能把知识讲明白，但很少真正帮助用户“学会”。

豆芽菜学习法的回答是：

> 学习不该止于解释，还应该包含复述、评估和巩固。

---

## 核心思想

这套方法的默认主流程只有四步：

1. 判断用户当前想要的是快速解释，还是进入学习模式
2. 生成结构化、低术语负担的学习笔记
3. 让用户用自己的话复述核心概念
4. 根据复述判断用户是否真的理解，并继续纠正或推进

在这个主流程之外，路线图、练习题、Anki 卡片、思维导图和学习档案都不是默认启动的，而是按需触发。

这让它既能保持轻量，也能支持长期系统学习。

---

## 它和普通学习 Prompt 的区别

- 它不把“讲完”当成“学会”
- 它把“复述”当成理解的核心检验
- 它区分快速解释、入门学习、系统学习、问题解决、深入研究等不同层级
- 它不默认让所有模块都上线，而是按用户需求触发
- 它可以跨平台迁移，而不是绑定某一个模型或产品

---

## 适合什么场景

- 用户第一次接触一个新概念，想真正理解
- 用户想系统学习一个主题，而不是零碎知道一点
- 用户在项目里遇到问题，想边解决边学习
- 用户已经有基础，想深入研究原理和边界
- 用户希望把学习变成可持续、可延续的过程

---

## 工作流结构

### 默认主流程

- 背景判断
- 结构化讲解
- 复述挑战
- 掌握度评估

### 按需触发的增强模块

- 学习路线图
- 刻意练习
- Anki 卡片
- 思维导图
- 学习档案

---

## 不同学习层级

这套方法默认支持以下层级：

- 快速解释：只想快速知道一个概念是什么
- 入门学习：第一次接触，想真正理解
- 系统学习：想建立完整知识体系
- 解决问题：围绕当前问题学习
- 深入研究：已有基础，想深挖原理、边界和细节
- 延续学习：基于之前的学习记录继续推进

---

## 为什么强调复述

因为“我看懂了”和“我真的会了”不是一回事。

很多学习型 AI 只负责把内容讲清楚，但真正决定用户是否掌握的，往往不是输入了多少，而是能不能把核心意思重新组织成自己的语言。

豆芽菜学习法把这一步放在中心位置：

> 解释只是开始，复述才是检验理解的关键。

## 如何使用

### 方式一：直接使用通用版提示词

把通用主提示词放进你使用的平台中，例如：

- ChatGPT Custom Instructions
- Claude Project Instructions
- Codex Skill / Agent Prompt

### 方式二：按平台使用适配版

如果不同平台的提示词长度、格式或触发机制不同，可以使用仓库内的专用版本。

### 方式三：继续迭代成自己的学习产品

你也可以把这套方法作为底层逻辑，继续扩展成：

- 垂直学科学习助手
- 课程陪练机器人
- 内容理解与复盘工具
- 带记忆和进度追踪的长期学习系统

---

## 当前版本方向

当前版本强调：

- 一个统一的方法，而不是多个割裂的 prompt
- 默认轻量主流程
- 按需触发增强模块
- 跨平台迁移能力

---

## 后续可扩展方向

- 做成 Codex 可安装 skill
- 提供 Claude / ChatGPT / Codex 的独立适配版
- 增加更多真实学习案例
- 增加流程图和产品说明
- 为不同学科设计专用变体

---

## 一句话总结

豆芽菜学习法不是让 AI “讲得更像老师”，而是让 AI 真正参与“帮助用户学会”这件事。
