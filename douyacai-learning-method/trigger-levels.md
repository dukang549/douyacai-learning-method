# 学习层级与触发规则

这个文档解释：豆芽菜学习法如何区分不同学习层级，以及每一层应由什么提示词或什么用户意图触发。

## 1. 快速解释

### 典型用户表达

- `X 是什么意思？`
- `简单解释一下 X`
- `用大白话讲 X`
- `让我快速知道 X 是什么`

### 系统应做什么

- 直接解释
- 不进入完整学习流程
- 不要求复述
- 不出路线图
- 不做 Anki

### 目标

帮助用户先知道“这是什么”。

## 2. 入门学习

### 典型用户表达

- `我想入门学习 X`
- `帮我理解 X`
- `带我学一下 X`
- `我想真正搞懂 X`

### 系统应做什么

- 进入主流程
- 生成轻量结构化笔记
- 默认讲 3 个左右核心点
- 发起复述
- 根据复述纠正

### 目标

帮助用户从“完全陌生”进入“基本理解”。

## 3. 系统学习

### 典型用户表达

- `我想系统学习 X`
- `给我一个学习路线来学 X`
- `从头到尾带我学 X`
- `帮我建立 X 的完整知识体系`

### 系统应做什么

- 触发路线图
- 按模块推进
- 每轮学习后进行复述检查
- 必要时输出思维导图
- 支持学习档案持续推进

### 目标

帮助用户建立完整知识结构，而不是零散知道一点。

## 4. 解决问题

### 典型用户表达

- `围绕这个问题教我 X`
- `我项目里遇到这个问题，帮我边解决边学`
- `我现在要用 X 解决一个具体问题`

### 系统应做什么

- 进入主流程
- 少铺大背景
- 多讲和当前问题直接相关的知识
- 以“能解决问题”为优先目标

### 目标

帮助用户把学习直接接到当前任务上。

## 5. 深入研究

### 典型用户表达

- `我想深入研究 X`
- `我已经懂基础了，想看原理`
- `从底层机制讲 X`
- `帮我深挖 X 的细节和边界`

### 系统应做什么

- 保留更多技术细节
- 强调原理、边界、比较和限制
- 允许引入论文、RFC、源码或前沿资料

### 目标

帮助用户从“会用”走向“理解得更深”。

## 默认判断规则

如果用户没有显式指定层级，优先按措辞自动判断：

- `是什么 / 简单解释` -> 快速解释
- `帮我理解 / 带我学` -> 入门学习
- `系统学习 / 路线 / 框架` -> 系统学习
- `解决这个问题 / 围绕项目教我` -> 解决问题
- `深入研究 / 原理 / 底层` -> 深入研究

如果用户明确指定层级，优先服从用户指定，不自行改写目标。

## 一句话原则

默认让 AI 判断层级，但同时保留清晰的用户触发词，这样既自然，也稳定。
