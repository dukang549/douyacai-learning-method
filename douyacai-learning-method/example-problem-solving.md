# Example: Problem Solving Learning

## User request

```text
我现在在做一个项目，别人总说要“调用 API”，但我其实不理解这个词。你围绕这个问题教我，不用上来讲一大堆背景。
```

## Expected mode

解决问题

## Why this mode

用户的目标不是建立完整体系，而是围绕当前项目理解一个足够能用的概念。

## Good response shape

```md
那我们就围绕“项目里为什么总要调用 API”来学，不先铺很大背景。

📚 学习主题：在项目里调用 API 到底是什么意思

## 核心概念
调用 API，本质上就是让你的程序去请求另一个系统提供数据或执行动作。
项目里之所以经常出现这个词，是因为很多能力并不是你自己从零实现，而是通过调用外部服务获得。

## 这次先掌握 3 个点
1. API 是软件之间的接口：你的程序可以通过它向别的系统提要求。
2. 调用 API 是一次请求动作：你发出去一个请求，对方返回结果。
3. 项目里常说“调接口”，通常就是在接入别人的能力，比如登录、支付、天气、AI。

## 一个例子
比如你做一个带 AI 总结功能的产品，不会自己训练模型，而是调用 AI 服务的 API，把文本发过去，再拿回总结结果。

🎯 现在轮到你了

请用自己的话解释：
1. 为什么项目里会频繁出现“调用 API”这个说法？
2. “调用 API”和“自己在本地写功能”有什么区别？
```

## What should not happen

- 不要默认切到系统学习
- 不要先讲 REST、GraphQL、OAuth 一整套大图景
- 不要离开用户当前问题太远

## Design note

这个案例用来说明：问题解决型学习更像“带着具体任务学”，它需要聚焦当前语境，而不是先建立完整知识体系。
